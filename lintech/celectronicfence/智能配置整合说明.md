# 智能配置功能整合说明

## 修改概述

将原有的独立"智能设置"页面功能整合到"任务管理"中，使每个任务可以独立配置对应算法的检测阈值。同时保留了原有智能设置的所有功能，包括批量操作。

## 修改内容

### 1. 任务管理页面增强 (task.html)

#### 新增功能

**1. 算法参数配置区域**
在任务创建/编辑界面中新增"算法参数"配置区块，动态阈值配置：
- **区域入侵**：人员检测阈值（默认 0.3）
  - ⚠️ **特殊功能保留**：
    - 输入 **1.0** = 一键关闭所有检测类别
    - 输入 **0** 或接近0的值 = 一键开启所有检测类别（设为0.3）
    - 输入 **0-1.0之间的值** = 只对person类别生效
- **人员检测**：人员检测阈值（默认 0.3）
- **明烟明火**：
  - 火焰检测阈值（默认 0.4）
  - 烟雾检测阈值（默认 0.4）

**2. 远程上报地址配置**
- 每个任务可以配置独立的远程上报地址
- 实现两层优先级机制：
  - **全局地址**（参数配置页面的"远程上报地址"）：优先级高
  - **任务独立地址**（任务编辑页面）：当全局地址为空时使用
- 启动任务时自动决策使用哪个地址

#### 界面改进
- 阈值数值在编辑界面中显示和配置，任务列表不显示具体数值
- 阈值输入框支持 0-1.0 范围，步长为 0.05
- 提供友好的配置提示信息，特别是区域入侵的特殊功能说明

#### 功能特点
- 每个任务独立配置阈值，互不影响
- 支持不同算法使用不同的阈值参数
- 编辑任务时自动加载已保存的阈值配置
- 启动任务时自动应用对应的阈值设置

### 2. 移除智能设置入口

#### index1.html (主界面)
- 注释掉侧边栏"智能设置"菜单项
- 注释掉对应的页面内容区域

#### home.html (预览界面)
- 注释掉侧边栏"智能设置"菜单项
- 注释掉对应的点击事件监听器

### 3. 保留文件

- `smart.html` 文件保留，但不再从界面入口访问
- 如需恢复功能，可以取消相关注释

## 优势

1. **更灵活**：每个任务可以设置不同的阈值，适应不同场景需求
2. **更直观**：在任务管理界面直接配置，无需切换页面
3. **更易用**：新增任务时就可以配置阈值，一步到位
4. **可扩展**：为未来支持更多算法参数预留了空间

## 使用说明

### 基本使用
1. 进入"任务管理"页面
2. 点击"新增任务"或编辑现有任务
3. 填写基本信息（任务编号、描述、视频源）
4. 选择算法后，会自动显示对应的阈值配置
5. 设置合适的阈值（建议范围 0.3-0.5）
6. （可选）配置此任务的远程上报地址
7. 保存任务

### 远程上报地址配置
#### 方式一：全局配置（推荐）
1. 进入"参数配置"页面
2. 找到"告警参数"分组
3. 编辑"远程上报地址"参数
4. 输入上报服务器地址（如：`http://192.168.1.100:8080/api/alarm`）
5. 保存参数
6. **所有任务启动时都会优先使用这个全局地址**

#### 方式二：任务独立配置
1. 在编辑任务界面
2. 填写"远程上报地址（可选）"字段
3. 此地址仅在全局地址为空时生效
4. 适用于不同任务需要上报到不同服务器的场景

#### 优先级说明
- ✅ **全局地址存在** → 使用全局地址（所有任务统一上报）
- ❌ **全局地址为空** → 使用任务自己的地址（灵活配置）

## 技术细节

### 数据结构

#### 区域入侵/人员检测任务：
```json
{
  "taskNumber": "TASK001",
  "description": "区域入侵检测",
  "videoSourceId": 1,
  "algorithm": "region_intrusion",
  "threshold": 0.3,
  "thresholdMode": "person_only",
  "remoteUrl": "http://192.168.1.100:8080/api/alarm",
  "status": "stopped"
}
```

**字段说明**：
- `threshold`：检测阈值（0-1.0）
- `thresholdMode`：阈值模式
  - `person_only`：只对person类别设置阈值（默认）
  - `enable_all`：一键开启所有检测类别（阈值0.3）
  - `disable_all`：一键关闭所有检测类别（阈值1.0）
- `remoteUrl`：任务独立的远程上报地址（可选）

#### 烟火检测任务：
```json
{
  "taskNumber": "TASK002",
  "description": "明烟明火检测",
  "videoSourceId": 1,
  "algorithm": "明烟明火",
  "flameThreshold": 0.4,
  "smokeThreshold": 0.4,
  "remoteUrl": "http://192.168.1.200:9090/fire/alarm",
  "status": "stopped"
}
```

### 阈值应用机制

1. **启动任务时自动设置阈值**：
   - 前端在调用 `/api/tasks/:id/start` 之前
   - 先调用 `http://192.168.1.231:8088/get_json_file` 设置阈值
   - 根据任务的 `thresholdMode` 决定设置方式

2. **区域入侵的三种模式**：
   - `person_only`：只设置person类别的阈值
   - `enable_all`：设置所有检测类别为0.3（一键开启）
   - `disable_all`：设置所有检测类别为1.0（一键关闭）

3. **批量操作功能保留**：
   - 用户输入 1.0 时，自动应用到所有检测类别（关闭）
   - 用户输入 0 或接近0的值时，自动应用0.3到所有类别（开启）
   - 用户输入正常值时，只应用到person类别

### 远程上报地址决策机制

启动任务时的地址决策流程：

```
启动任务
   ↓
获取全局参数 (GET /api/params)
   ↓
判断全局RemoteInfo是否有值？
   ↓
 有值 ──→ 使用全局地址
   ↓
 无值 ──→ 使用任务的remoteUrl
   ↓
启动任务 (POST /api/tasks/:id/start)
传递最终决定的remoteUrl
```

**实现逻辑**：
```javascript
function determineRemoteUrl(globalUrl, taskUrl) {
    // 优先使用全局地址
    if (globalUrl && globalUrl.trim() !== '') {
        return globalUrl.trim();
    }
    // 全局为空时使用任务地址
    return taskUrl && taskUrl.trim() !== '' ? taskUrl.trim() : '';
}
```

### API 支持

#### 前端调用的API：
1. **设置阈值**：`POST http://192.168.1.231:8088/get_json_file`
2. **获取全局参数**：`GET /api/params`
3. **启动任务**：`POST /api/tasks/:id/start`（需要携带remoteUrl）
4. **保存任务**：`POST /api/tasks` 和 `PUT /api/tasks/:id`

#### 后端需要支持的字段：

**任务数据字段**：
- `threshold` 和 `thresholdMode`（区域入侵/人员检测）
- `flameThreshold` 和 `smokeThreshold`（烟火检测）
- `remoteUrl`（任务独立的上报地址）

**全局参数字段**：
- `RemoteInfo`（全局上报地址，位于参数配置中）

#### 后端建议实现：
启动任务时接收前端传递的 `remoteUrl`，并在告警上报时使用该地址。
